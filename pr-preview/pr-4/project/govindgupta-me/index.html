Overview

This portfolio was designed to feel quiet, sharp, and durable. I wanted a site that could hold projects, writing, and personal context without becoming visually noisy or overly decorative.

Problem Statement

Most developer portfolios either feel too sparse to be memorable or too busy to be useful. The challenge here was to create a site with strong hierarchy, real personality, and enough flexibility to grow with new writing, case studies, and future work.

Solution and Technical Decisions

I built the site with the App Router so pages, metadata, and content could stay close together. MDX powers long-form writing and project case studies, while the visual system stays intentionally restrained so typography and spacing carry most of the identity.

Key Features

The final product combines live GitHub data, theme support, blog rendering, project case studies, and custom Open Graph generation. Each addition was shaped to feel like part of one system rather than a collection of unrelated features.

Learnings

Working on a personal site always exposes where design taste and engineering discipline meet. The best improvements were usually the ones that removed noise, clarified content, and made the structure easier to extend later.
